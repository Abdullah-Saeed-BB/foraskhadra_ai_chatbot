# روبوت الدردشة الذكي لفرص خضراء
روبوت محادثة ذكي ومستشار توظيف مؤتمت (بأسم خبير)، مصمم لمساعدة المستخدمين على استكشاف وإيجاد فرص العمل والتطوع التي تناسب مهاراتهم وخبراتهم مباشرة عبر منصة **"فرص خضراء"**. يعمل النظام كوكيل ذكي (AI Agent) قادر على تحليل طلبات المستخدمين المعقدة، واستخراج النوايا، وتصفية البيانات بدقة تامة لتقديم أفضل الفرص المتاحة من دون اي هلوسات.

## 🚀 مميزات المشروع الرئيسية (Key Features)
### البحث الدلالي الذكي (Hybrid RAG & Filtering)
دمج تقنية الاسترجاع المدعم بالتعزيز (RAG) مع أنظمة التصفية الديناميكية (Filtering) لضمان جلب الفرص الأكثر ملاءمة لمهارات وخلفية المستخدم, مع عرض الفرص المسترجعة على هيئة بطاقات مرئية منظمة وجذابة لتعزيز تجربة المستخدم (UI/UX).

### اقتراحات الأسئلة الذكية (Prompt Suggestions):
توليد واقتراح أسئلة ديناميكية لإلهام المستخدم ومساعدته في استكشاف المنصة والوصول إلى الفرص المتاحة بسلاسة.

## 🏗️ البنية التقنية والتقنيات المستخدمة (Tech Stack)

تم بناء المشروع بأسلوب هندسي يعتمد على فصل المهام ومعالجة البيانات الموزعة لضمان سرعة الاستجابة وكفاءة الأداء:
### 1. إدارة وتوجيه الذكاء الاصطناعي (AI Orchestration)
*   **إطر عمل LangGraph & LangChain:** لبناء هيكلة الوكيل الذكي (AI Agent Architecture)، وإدارة الحالات (State Management)، وتوجيه تدفق المحادثة بشكل دقيق وسلس.

### 2. نماذج اللغة الكبيرة (LLMs)
تم اعتماد استراتيجية توجيه المهام لعدة نماذج عبر منصة **Groq** لتحسين الأداء وتقليل التكلفة:
*   **المهام المعقدة:** الاعتماد على نموذج `GPT OSS 120B` للتحليل العميق والإجابة على اسئلة المستخدمين.
*   **المهام البسيطة:** الاعتماد على نموذج خفيف مثل `GPT OSS 20B` للإجابة على الاستفسارات البسيطة وتوليد الاقتراحات.

### 3. طبقة الواجهة البرمجية (Backend API)
* **إطار عمل FastAPI:** بناء خادم برمجيات يعتمد على معالجة الطلبات غير المتزامنة (Asynchronous)، لربط واجهة المستخدم بوكيل الذكاء الاصطناعي وقواعد البيانات.

### 4. الواجهة الأمامية (Frontend)
*   **إطار عمل Next.js:** لإعداد تطبيق ويب سريع متوافق مع معايير الـ SSR (Server-Side Rendering).
*   **مكتبة Tailwind CSS:** لبناء وتصميم الواجهات الرسومية وتنسيق الصفحات و بطاقات عرض الوظائف بأسلوب عصري مرن.

## 📊 آلية عمل النظام (System Flow)

1. **الاستقبال وفهم النية:** يستقبل الوكيل سؤال المستخدم عبر الـ API المعالج بواسطة FastAPI.
2. **توجيه النموذج (Routing):** يحدد النظام ما إذا كان السؤال يتطلب تحليلاً عميقاً أو إجابة عامة سريعة لتوجيهه للنموذج المناسب على Groq.
3. **استرجاع البيانات (RAG):** يتم البحث دلالياً في الفرص المتاحة بمنصة "فرص خضراء" وتطبيق فلترة مخصصة بناءً على طلب المستخدم.
4. **العرض الجمالي:** تُرسل البيانات المسترجعة للواجهة الأمامية ليقوم Next.js بصياغتها على شكل بطاقات تفاعلية مريحة للعين.
